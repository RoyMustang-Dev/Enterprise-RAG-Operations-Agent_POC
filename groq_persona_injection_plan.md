
## Caching Strategy (Performance Fix):
Currently, SQLite queries add latency. To execute this seamlessly, the Backend will use an **In-Memory Cache Singleton**. The Persona payload is fetched from your DB *exactly once* on server startup and loaded into RAM. All 12 node routing operations fetch the persona from RAM instantly (`0ms`), only touching SQLite again if a user explicitly clicks "Save New Agent" to invalidate the cache.

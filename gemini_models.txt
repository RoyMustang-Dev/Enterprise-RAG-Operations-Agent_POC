================================================================================
ENTERPRISE RAG ARCHITECTURE: 12-STEP GEMINI HYPER-TUNING & DEEP RESEARCH REPORT
================================================================================
Based on deep source-code inspection of the existing Groq-based architecture, the current constraints were identified:
- Groq's micro-models (8B) struggled with JSON strictness and over-hallucinated specific intent metadata (e.g. creating overly specific metadata filters that caused vector search to return 0 results).
- The "Smalltalk" agent was literally a hardcoded fallback string ("Hello! I am the Enterprise RAG System..."), which resulted in awful UX (the "bullshit" response mentioned).
- Groq's 30-sec timeout boundaries severely restricted the Core Synthesis engine from doing deep Document-Cross-Reference Verification.

By migrating to the Gemini API (which features 1M+ context limits, natively optimized ReAct structured outputs, and integrated Multimodal capabilities), we can radically enhance every single node!

--------------------------------------------------------------------------------
PART 1 & 2: MODEL MAPPING & OPTIMIZED ARCHITECTURE REFACTORING
--------------------------------------------------------------------------------

1. Prompt Injection Guard (app/prompt_engine/guard.py)
   -> Model Chosen: `models/gemini-2.5-flash-lite`
   -> Fixes: Groq models suffered latency spikes parsing nested payload structures. Gemini Flash Lite has near-zero latency and built-in Google SafeSearch guardrails, allowing hyper-fast intercepting of prompt injections without spending expensive compute.

2. Intent Classification / Routing (app/supervisor/intent.py)
   -> Model Chosen: `models/gemini-2.5-flash`
   -> Fixes: Replaces the fragile Groq 8B routing dictionary. Gemini 2.5 Flash natively outputs guaranteed JSON Schema. Instead of just picking single intents, the ReAct prompt allows the model to map out a "Multi-Step Execution Plan" immediately.

3. Smalltalk & Analytics Agent (app/supervisor/router.py)
   -> Model Chosen: `models/gemini-2.0-flash-001`
   -> Fixes: REPLACES the hardcoded string. We now route non-RAG chat to a dedicated, empathetic, and memory-aware conversation agent that maintains brand voice instead of a robot fallback.

4. Prompt Rewriter & Query Expansion (app/prompt_engine/rewriter.py)
   -> Model Chosen: `models/gemini-2.5-flash`
   -> Fixes: Instead of blindly expanding words, we utilize Gemini's context window to inject the *User Chat History* into the rewriter, creating semantically perfect query expansions (creating 3 canonical search variations) that fix the "Metadata Filters were too specific" bug.
   -> Additions: we want to save this "Prompt Rewriter & Query Expansion" steps' output "the re-written/expanded prompt" along with the original (user's prompt/query) in SQLite DB, along with other necessary things.


5. Metadata Filter Extraction (app/retrieval/metadata_extractor.py)
   -> Model Chosen: `models/gemini-2.5-pro`
   -> Fixes: Groq hallucinated strict IDs. By using Gemini Pro's ToT (Tree-of-Thoughts) reasoning, we execute "Broad-to-Narrow" filter generation. It knows to provide relaxed fuzzy filters rather than exact matching string constraints, guaranteeing optimal vector DB retrieval.

6. Vision / OCR Grounding (Multimodal Ingestion)
   -> Model Chosen: `models/gemini-2.5-pro`
   -> Fixes: Bypasses the clunky standalone LLaVA/BLIP integrations. Images bypass OCR entirely and are natively ingested by Gemini 2.5 Pro which extracts visual charts, spatial text, and hidden graphs effortlessly.

7. Audio / Voice Ingestion (Multimodal STT)
   -> Model Chosen: `models/gemini-2.5-flash-native-audio-latest`
   -> Fixes: Replaces standalone Whisper models. Gemini native audio processes raw audio bytes for precise emotional and phonetic semantic analysis.

8. Query Complexity Assessor (app/reasoning/complexity.py)
   -> Model Chosen: `models/gemini-2.5-flash-lite`
   -> Fixes: Instantly maps if a query needs standard retrieval or Deep Research (Agentic Loop). 

9. Vector Semantic Retrieval & Reranking
   -> Model Chosen: `models/gemini-embedding-001`
   -> Fixes: Translates the user query into high-dimensional vectors tailored specifically to the structural intent mapped by Step 2.

10. Core RAG Dense Synthesis (app/reasoning/synthesis.py)
    -> Model Chosen: `models/gemini-3.1-pro-preview` (Frontier)
    -> Fixes: Groq 70B struggled with context degradation over 8K tokens. Gemini 3.1 Pro easily processes 1M+ tokens in a single shot. I've designed a specialized Chain-of-Thought (CoT) prompt that forces the model to cite the Exact Document IDs before writing the synthesized answer.

11. Verification & Hallucination Check (app/reasoning/verifier.py)
    -> Model Chosen: `models/gemini-2.0-flash`
    -> Fixes: Completely isolates the Verification graph from the Synthesis graph to avoid confirmation bias. Verifier is tasked purely to find contradictions between the Context Chunks and the Synthesized Output. 

12. RLHF Data Grader / Reward Model (app/rlhf/reward_model.py)
    -> Model Chosen: `models/gemma-3-27b-it`
    -> Fixes: An internal self-improvement loop. Gemma 3 27B runs asynchronously after the user gets the answer to score the pipeline's execution speed, accuracy, and safety, appending data for future few-shot alignments without incurring cloud API costs.

PART 5: MAIN LLM BRAIN CLARIFICATION
--------------------------------------------------------------------------------
Q: Which model is the Main LLM Brain and Why?
A: `models/gemini-3.1-pro-preview` (for complex logic) and `models/gemini-2.5-flash` (for fast execution). 
- Why: The 3.1 Pro preview is Google's absolute frontier reasoning model. It replaces the old Groq Llama 3 70B entirely. It has a context window large enough to ingest entire codebases or 50+ PDF documents simultaneously, enabling deep semantic grounding that completely eliminates hallucinations.

Q: Can this main LLM brain natively process all the file uploads?
A: 
- .pdf, .jpg, .jpeg, .png, .txt, .md, .csv, .tsv = YES. Natively, perfectly, directly via the API Payload without external helpers.
- .docx, .xlsx = NO (Directly). The server will need a 1-second Python parser to extract the raw text/tables from these zipped XML formats, but once extracted, the LLM Brain devours the output flawlessly.

PART 6: "CREATE NEW AGENT" PERSONA BOOTSTRAPPER & NATIVE AUDIO INTEGRATION
--------------------------------------------------------------------------------

1. The "Create New Agent" Flow (FastAPI Swagger Endpoint) 
   -> This must be placed at the very top of the Swagger's UI, meaning as the first endpoint.
   -> Endpoint: `POST /api/v1/agents`
   -> Content-Type: `multipart/form-data` (Required for handling text fields + image file simultaneously).
   -> Must be global, meaning it will affect all the backends 'groq-core', 'gemini-core', 'anthropic-core', 'openai-core'.
   -> Swagger UI Inputs & Instructions:
      - `bot_name` (Text): The name of your AI Assistant (e.g., "AekoBot").
      - `company_logo` (File): Upload a `.png` file representing the brand (e.g., `company_name.png`). Uniquely name it if updating.
      - `brand_details` (Text): Example -> "An EdTech Platform, Empowering the next generation of data scientists and AI specialists through accessible, high-quality education."
      - `welcome_message` (Text): The very first text bubble the user sees in the UI *before* typing. (e.g., "Hi! I am {Bot Name}, here to help you navigate our Enterprise Data. How can I assist you today?")
      - `prompt_instructions` (Text) [Optional]: Provide detailed instructions on how the bot should behave. Example: "You are a highly analytical AI. Always respond in markdown tables when comparing data."

   -> Backend Execution Flow:
      1. Receives the `multipart/form-data`. Saves the `company_logo` locally securely (e.g., `app/static/logos/akobot.png`).
      2. Persona Expansion Hook: Takes the user's `prompt_instructions` (if provided) and passes it to the `Prompt Rewriter` (Gemini 2.5 Flash). Gemini transforms the base instructions into a hyper-detailed, ReAct-compliant `expanded_prompt`.
      3. Persistent Storage (SQLite): Saves the entire multitenant profile (`bot_name`, `logo_path`, `brand_details`, `welcome_message`, `raw_prompt`, `expanded_prompt`, `creator_user_id`) into a dedicated Enterprise SQL table (e.g., `agent_profiles`). This is vastly superior and more secure than raw text logging.
      4. Universal Injection: The `expanded_prompt` (Persona) is now dynamically fetched by the Router and appended to the System Prompts of ALL downstream nodes (Smalltalk Agent, Coder Agent, RAG Synthesis) so the brand voice never breaks.

2. Live Real-Time Conversation (Multimodal STT/TTS)
   -> Model: `models/gemini-2.5-flash-native-audio-latest`
   -> Capabilities: Natively supports WebRTC. Treats human speech and interruptions flawlessly in real-time. Bypasses text-bottlenecks.
   -> Persona Auto-Injection: The exact `expanded_prompt` generated above is natively pulled from the SQLite DB and injected as the `system_instruction` parameter when initializing the Live Audio session.

   -> Real-Time Audio System Prompt (Hyper-Tuned Gemini ReAct):
      ```text
      SYSTEM: You are the Live Vocal Interface dynamically mapped to {Bot_Name}.
      <Brand_Details> {Brand_Details} </Brand_Details>

   -> Real-Time Session Triggers & Voice Welcome Message:

        ```
      - This ensures the model natively speaks a contextual, brand-aligned greeting the absolute millisecond the microphone stream connects, creating a flawless user experience.
